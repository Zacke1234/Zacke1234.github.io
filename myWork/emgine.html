Introduction

Having to build up a engine from scratch is very difficult but doable! With the help of our teachers & the futuregames course I am able to make this cool little engine.

Engine




Role: Programmer

Time: Ongoing

My game engine named "Emgine" is part of our game engine specialisation at futuregames.

It's built on C++ with several github plugins like Khronos, Dear imgui and OpenGL.

Emgine

Programming

The system of my game engine goes like this: There's classes for drawing cubes, a class for drawing meshes that it takes from a file, there's an virtual object class that calls to the cube and object loader class. It also got it's own class for the UI, which uses the Imgui plugin. The engine has it's main.cpp file where every other class gets called in. I use a VertexShader and Fragmant shader files for the rendering of my engine. There is lots of going back and forth with the code cause of the little and small things you have the implement for your engine, it is crucial that you learn how datastructures, object oriented programming work for a well functional engine code database!
